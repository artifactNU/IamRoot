# Linux Basics

This directory contains fundamental Linux concepts that every system administrator should understand.

The focus is on **core system knowledge** that remains stable across distributions and versions.

---

## Topics Covered

- **Filesystem Hierarchy** - Where things live and why
- **Permissions and Ownership** - Access control fundamentals
- **Processes** - How programs run and how to manage them
- **System Logs** - Where to look when things go wrong
- **Users and Groups** - Identity and access management
- **Package Management** - Installing and maintaining software

---

## Audience

These guides assume:
- Basic command-line familiarity
- Some system administration exposure
- Ubuntu/Debian environment (concepts translate to other distros)

They do not assume deep kernel knowledge or years of experience.

---

## Philosophy

Documentation here should:
- Explain *why*, not just *what*
- Address common failure modes
- Connect concepts to real operational problems
- Age well as systems evolve

If you are looking for quick command references, see the `cheat-sheets/` directory instead.
