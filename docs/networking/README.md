# Networking

This directory contains **networking fundamentals and troubleshooting guides** for Linux system administration.

The focus is on **practical network operations**, diagnostics, and understanding how traffic flows through systems.

---

## Topics Covered

- **TCP/IP Fundamentals** - How the network stack works
- **Network Interfaces** - Configuration and management
- **Routing** - How packets find their destination
- **DNS and Name Resolution** - How names become addresses
- **Firewalls** - Controlling network access
- **Network Troubleshooting** - Tools and techniques for diagnosing issues
- **Common Problems** - SSH issues, slow connections, port conflicts

---

## Audience

These guides assume:
- Basic understanding of IP addresses and networks
- Familiarity with command-line tools
- Some exposure to network concepts

They do not assume deep protocol knowledge or network engineering experience.

---

## Philosophy

Documentation here should:
- Explain how traffic flows through the system
- Focus on operational diagnostics
- Address common failure modes
- Provide working examples, not just theory
- Connect networking concepts to real problems

If you need quick command references, see the `cheat-sheets/` directory instead.

---

## When to Use These Guides

- Service is unreachable
- SSH connection issues
- DNS resolution failures
- Firewall blocking traffic
- Port conflicts
- Network performance problems
- Understanding routing decisions

The goal is to help you answer: "Why can't this machine talk to that one?"
