# Security

This directory contains **security hardening and auditing guides** for Linux system administration.

The focus is on **practical security measures** that can be implemented and verified on production systems.

---

## Topics Covered

- **System Hardening** - Reducing attack surface and securing configurations
- **User and Access Control** - Managing authentication and authorization
- **Auditing and Monitoring** - Detecting suspicious activity
- **Incident Response** - What to do when something goes wrong
- **SSH Security** - Securing remote access
- **File Integrity** - Detecting unauthorized changes

---

## Audience

These guides assume:
- System administration experience
- Understanding of Linux permissions and processes
- Familiarity with basic security concepts

They do not assume deep security expertise or penetration testing skills.

---

## Philosophy

Documentation here should:
- Provide actionable security improvements
- Explain why measures matter, not just what to do
- Focus on realistic threats for typical environments
- Balance security with operational needs
- Be implementable by sysadmins, not just security specialists

## When to Use These Guides

- Setting up new systems
- Hardening existing infrastructure
- Responding to security incidents
- Conducting security audits
- Investigating suspicious activity
- Implementing compliance requirements

The goal is to help you answer: "Is this system secure enough, and how do I know?"
