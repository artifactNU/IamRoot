# systemd Configuration Examples

Example systemd unit files and configurations.

---

## Purpose

Provide reference templates for:
- Service units
- Timer units (cron replacements)
- Mount units
- Common service patterns

---

## Guidelines

All examples must:
- Use `.example` suffix
- Include inline documentation
- Follow systemd best practices
- Be compatible with systemd 245+ (Ubuntu 20.04)
- Avoid hardcoded paths where possible

---

## Example Topics

Future examples may include:
- Simple service unit
- Service with restart policies
- Timer unit for periodic tasks
- Socket-activated service
- User service units
